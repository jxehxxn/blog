# Gemini CLI의 핵심, MCP(Managed Code Piper)란?

Gemini CLI의 가장 강력하고 독창적인 기능은 바로 **MCP(Managed Code Piper)**입니다. MCP는 Gemini가 단순한 언어 모델을 넘어, 다른 소프트웨어를 직접 제어하고 상호작용할 수 있도록 만들어주는 '다리' 역할을 합니다.

## MCP의 개념

MCP는 Gemini CLI와 외부 애플리케이션(예: 웹 브라우저, IDA Pro) 사이에 위치하는 경량 서버 또는 플러그인입니다. 작동 방식은 다음과 같습니다.

1.  **사용자 명령어 입력**: 사용자가 "구글에 접속해서 'Gemini CLI'를 검색해줘" 와 같이 자연어로 명령을 내립니다.
2.  **Gemini의 도구 사용 판단**: Gemini 모델은 이 명령을 수행하기 위해 Playwright MCP(웹 브라우저 제어용)를 사용해야 한다고 판단합니다.
3.  **MCP에 명령어 전달**: Gemini는 "google.com으로 이동", "'Gemini CLI' 입력", "검색 버튼 클릭" 등과 같은 구체적인 제어 명령어를 생성하여 Playwright MCP에 전달합니다.
4.  **애플리케이션 제어**: MCP는 Gemini로부터 받은 명령어를 해당 애플리케이션이 이해할 수 있는 API 호출(예: Playwright의 `page.goto()`, `page.fill()`)로 변환하여 실행합니다.
5.  **결과 반환**: 애플리케이션에서 수행된 결과(예: 웹페이지의 내용, 스크린샷)는 다시 MCP를 통해 Gemini에게 전달됩니다.
6.  **사용자에게 최종 결과 보고**: Gemini는 전달받은 결과를 바탕으로 사용자의 초기 질문에 대한 최종 답변을 생성합니다.

## 왜 MCP가 중요한가?

MCP는 Gemini의 능력을 무한히 확장시킵니다. MCP를 통해 Gemini는 다음과 같은 일들을 할 수 있게 됩니다.

*   **GUI 애플리케이션 자동화**: 웹 브라우저, 리버스 엔지니어링 도구 등 그래픽 인터페이스를 가진 프로그램을 자연어로 제어합니다.
*   **특화된 작업 수행**: 범용적인 AI 모델이 직접 수행하기 어려운 전문 분야의 작업(예: 악성코드 정적 분석, 웹사이트 E2E 테스트)을 해당 분야의 전문 도구를 통해 수행합니다.
*   **상태 기반의 연속 작업**: 단순한 일회성 명령이 아니라, "로그인하고 -> 대시보드로 이동해서 -> 특정 데이터를 가져와" 와 같이 여러 단계에 걸친 복잡한 작업을 상태를 유지하며 수행할 수 있습니다.

## 현재 사용 가능한 주요 MCP

Gemini CLI 공식 레포지토리에서는 다음과 같은 MCP를 지원하거나 예시를 제공하고 있습니다.

*   **Playwright MCP**: 웹 브라우저 자동화 라이브러리인 Playwright를 제어하여 웹 스크래핑, E2E 테스트, 웹 기반 작업 자동화 등을 수행합니다.
*   **IDA-MCP**: 리버스 엔지니어링의 표준 도구인 IDA Pro와 연동하여 바이너리 파일 분석, 디컴파일, 스크립팅 작업을 자연어로 수행합니다.
*   **OS-MCP**: 운영체제 수준의 작업을 수행하기 위한 MCP로, 파일 시스템 제어, 프로세스 관리 등을 더 지능적으로 수행할 수 있습니다.

다음 포스트부터는 실제로 Playwright MCP와 IDA-MCP를 어떻게 설정하고 활용하는지 구체적인 예시와 함께 알아보겠습니다.
