## 1. 비유 — "Prometheus를 그대로 logs로"

Loki는 "Prometheus inspired 로그 시스템". label 기반 인덱스 + 본문은 object storage. 비용 압도적 저렴.

ELK는 본문까지 모두 인덱싱 → 강력하지만 비쌈.

## 2. 아키텍처

```
[ App ] → stdout
       ↓
[ Promtail / Fluent Bit DaemonSet ] (각 노드)
       ↓ push
[ Loki distributor ]
       ↓
[ ingester ] → S3 (chunks)
       ↓
[ querier ] ← S3
       ↓
[ Grafana ] → LogQL
```

각 컴포넌트가 microservice. monolithic mode (단일 binary) 도 있음.

## 3. 설치

```bash
helm install loki grafana/loki-stack -n monitoring
# Loki + Promtail
```

production: grafana/loki helm chart로 micro service 모드.

## 4. Promtail vs Fluent Bit

- Promtail: Loki 전용. label scraping K8s 친화.
- Fluent Bit: 범용. Loki/ES/CloudWatch 등 multi destination.

빅테크 multi-backend 환경 → Fluent Bit.

## 5. Label 설계 — 가장 중요

Loki는 label index 기반. label cardinality 폭발 시 비용·성능 폭증.

**권장 label**:
- namespace
- app / job
- container_name
- pod (조심)
- level (info/warn/error)

**금지 label**:
- user_id
- request_id
- timestamp

Cardinality 깊은 분석은 [보충 3: Logs cardinality].

## 6. LogQL 기본

### Log query
```logql
{app="myapp", level="error"}
```

label matcher로 stream 선택.

### Filter
```logql
{app="myapp"} |= "exception"
{app="myapp"} != "DEBUG"
{app="myapp"} |~ "5\\d\\d"
```

`|=` 포함, `!=` 미포함, `|~` regex.

### Parse
```logql
{app="myapp"} | json | latency > 1
{app="myapp"} | logfmt | status="500"
{app="myapp"} | regexp "user=(?P<user>\\w+)" | user="alice"
```

JSON/logfmt 자동 parse 후 label로 사용.

### Metric query
```logql
sum by (level) (rate({app="myapp"}[5m]))
```

PromQL과 비슷. log를 metric으로.

## 7. 실전 예시

### 에러 비율
```logql
sum(rate({app="payments"} |= "ERROR" [5m]))
/
sum(rate({app="payments"}[5m]))
```

### Slow request
```logql
{app="payments"} | json | latency > 5
```

### Top error sources
```logql
topk(5,
  sum by (pod) (rate({app="payments"} |= "ERROR" [5m]))
)
```

## 8. Retention

```yaml
limits_config:
  retention_period: 30d
```

Compactor가 주기적 cleanup.

빅테크 패턴: hot (30d) on hot storage + cold (1y) on glacier 등 tiered.

## 9. Grafana 통합

Grafana datasource로 Loki 추가. Prometheus와 같은 dashboard에 logs 패널 추가.

특히 강력: metric 보다가 spike 발견 → 클릭으로 같은 시각 logs 자동 조회 ("Explore" 모드).

## 10. 운영 함정 5선

1. **고 cardinality label**: 비용 폭발.
2. **retention 너무 길게**: S3 비용.
3. **Promtail의 추가 parsing 무거움**: 노드 CPU 부담.
4. **Loki 단일 모드 + 큰 트래픽**: 성능 부족.
5. **Grafana에서 LogQL 잘못 (label matcher 없이 |= "x")**: 전체 scan으로 query timeout.

## 11. 빅테크 사례

### Grafana Labs
Loki 모기업. 자체 SaaS.

### Spotify
ELK → Loki 부분 이주로 cost 1/10.

### 일부 빅테크
ELK 유지 (full-text search 필수 요건 시).
