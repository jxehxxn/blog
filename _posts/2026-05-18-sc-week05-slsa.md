## 학습 목표

- SLSA 4 레벨의 정확한 요구.
- 각 레벨 달성 도구.
- 빅테크 표준.

## 1. SLSA = Supply chain Levels for Software Artifacts

Google 시작 → OpenSSF 표준. v1.0 정식.

## 2. Level 1: Provenance Available

요구: 빌드가 어디서 누가 어떻게 만들어졌는지 메타데이터(provenance) 존재.

도구: 어떤 CI라도 manual 작성 가능.

거의 모든 빌드 가능. 시작점.

## 3. Level 2: Hosted Build + Signed Provenance

요구:
- 빌드를 hosted service (GitHub Actions, GitLab, CircleCI 등)에서.
- provenance가 서명됨.

도구: GitHub Actions + cosign + slsa-github-generator.

## 4. Level 3: Non-falsifiable

요구:
- 빌드 인프라가 신뢰됨 (격리).
- provenance가 빌드 서비스 자체가 서명 (사용자 키 X).
- 사용자가 provenance 위조 불가.

도구: SLSA L3 GitHub generator, Tekton Chains.

빅테크 표준.

## 5. Level 4: Two-person Review + Hermetic + Reproducible

요구:
- 모든 코드/빌드 변경에 2인 리뷰.
- Hermetic: 빌드가 외부 의존성 없음 (네트워크 차단).
- Reproducible: 같은 input → bit identical output.

도구: Bazel (hermetic), 자체 시스템.

극소수만 달성. Google 일부 internal.

## 7. SLSA Verification

```bash
slsa-verifier verify-image mycorp/myapp:v1.0.0 \
  --source-uri github.com/mycorp/myapp
```

## 8. 도구 → Level 매핑

| 도구 | 달성 |
|------|------|
| GitHub Actions + manual | L1 |
| GitHub Actions + slsa-generator | L3 |
| Tekton Chains | L3 |
| Bazel + hermetic | L4 후보 |

## 9. 운영 함정

1. provenance 생성만, 검증 누락 → 의미 없음.
2. Hermetic 어려움 (대부분 build가 network 사용).
3. Reproducible: timestamp, random 제거 필요.
4. L3 generator 잘못 사용 → 실질 L2.
