반갑습니다. 글로벌 빅테크 기업에서 30년 넘게 정보 보안의 최전선을 지켜온 전문가입니다. 

수많은 보안 솔루션이 명멸하는 현장에서 제가 가장 신뢰하고, 또 실무자들에게 강력하게 추천하는 도구가 바로 **Splunk**입니다. 단순히 '로그 분석 툴'이라고 부르기엔 그 잠재력이 너무나 거대하죠. 오늘부터 12주 동안, 여러분을 단순한 사용자가 아닌 '데이터를 지배하는 보안 분석가'로 만들어 드릴 여정을 시작합니다.

이 과정은 이론에 치우치지 않습니다. 실제 빅테크 환경에서 마주하는 거대한 데이터의 파도를 어떻게 제어하고, 그 안에서 위협의 흔적을 찾아내는지 실무적인 관점에서 풀어낼 것입니다.

## 🚀 1주차: 로그는 시스템의 DNA다

### 1. 빅테크에서 '로그'가 갖는 의미

많은 입문자가 로그를 단순히 "시스템이 뱉어내는 텍스트 파일" 정도로 생각합니다. 하지만 빅테크 보안의 관점에서 로그는 **시스템의 DNA**이자 **디지털 발자국**입니다.

수만 대의 서버와 마이크로서비스가 얽혀 돌아가는 환경에서, 어떤 사용자가 언제, 어디서, 어떤 경로로 데이터에 접근했는지 증명할 수 있는 유일한 수단은 로그뿐입니다. 로그가 없다면 우리는 눈을 감고 전장을 누비는 것과 같습니다. Splunk는 이 파편화된 DNA 조각들을 모아 하나의 생생한 이야기로 재구성해주는 도구입니다.

### 2. Splunk의 철학: "로그를 위한 구글"

Splunk의 핵심 철학은 명확합니다. **"어떤 데이터든, 어떤 형태든, 일단 다 집어넣어라. 그리고 구글처럼 검색해라."**

기존의 관계형 데이터베이스(RDBMS)는 데이터를 넣기 전에 미리 칸(Schema)을 짜야 했습니다. 하지만 로그는 형식이 제각각이고 시시각각 변합니다. Splunk는 데이터를 저장할 때 형식을 따지지 않습니다. 대신 검색하는 순간에 의미를 부여합니다(Schema-on-the-fly). 이 유연함이 바로 Splunk를 빅데이터 분석의 왕좌에 앉힌 비결입니다.

### 3. Splunk 아키텍처: 완벽한 분업 시스템

Splunk의 구조를 이해하기 위해 **'거대 도서관'**의 비유를 들어보겠습니다. Splunk는 크게 세 가지 핵심 컴포넌트로 구성됩니다.

#### ① Forwarder (전달자) - "도서 수집 운반팀"
각 서버(데이터 소스)에 상주하며 발생하는 로그를 실시간으로 수집해 인덱서로 보내는 역할을 합니다. 
- **Universal Forwarder (UF):** 아주 가볍고 최소한의 기능만 수행하는 운반 트럭입니다.
- **Heavy Forwarder (HF):** 데이터를 보내기 전에 미리 분류하거나 필터링할 수 있는 지능형 운반 차량입니다.

#### ② Indexer (인덱서) - "사서와 서고"
Forwarder가 보내온 데이터를 받아서 저장하고, 나중에 빨리 찾을 수 있도록 색인(Index)을 만듭니다.
- 데이터를 '버킷(Bucket)'이라는 단위로 쪼개어 저장합니다.
- 검색 요청이 들어오면 자신의 서고를 뒤져 결과를 내놓습니다.

#### ③ Search Head (검색 헤드) - "수석 연구원"
사용자가 검색창에 명령어를 입력하는 인터페이스입니다.
- 사용자의 요청을 해석해서 여러 대의 인덱서에게 "이 데이터 좀 찾아줘"라고 명령을 내립니다.
- 인덱서들이 보내온 결과물을 취합해서 예쁜 그래프나 표로 시각화합니다.

## 📝 1주차 자기진단 퀴즈 (Self-Assessment)

학습한 내용을 점검해봅시다. 정답은 아래에 있습니다.

**Q1. Splunk 아키텍처 중, 원본 데이터를 수집하여 전송하는 역할을 담당하는 컴포넌트는 무엇인가요?**
1) Search Head
2) Indexer
3) Forwarder
4) Deployment Server

**Q2. Splunk의 'Schema-on-the-fly' 개념에 대한 설명으로 옳은 것은 무엇인가요?**
1) 데이터를 저장하기 전에 미리 테이블 구조를 정의해야 한다.
2) 데이터를 검색하는 시점에 필드와 의미를 추출한다.
3) 실시간 데이터만 처리할 수 있고 과거 데이터는 처리하지 못한다.
4) 오직 JSON 형식의 데이터만 처리할 수 있다.

**Q3. 빅테크 환경에서 로그 분석이 보안의 핵심인 이유는 무엇인가요?**
1) 로그 파일의 용량이 크기 때문에 저장 공간을 확보하기 위해서
2) 시스템의 모든 활동을 기록한 유일한 증거(DNA)이기 때문에
3) 프로그래밍 언어를 배우지 않아도 되기 때문에
4) 서버의 하드웨어 사양을 확인하기 위해서

### 💡 정답 및 해설

**A1. 정답: 3) Forwarder**
*해설: Forwarder는 데이터 소스에서 로그를 수집하여 Indexer로 전달하는 '운반팀' 역할을 합니다.*

**A2. 정답: 2) 데이터를 검색하는 시점에 필드와 의미를 추출한다.**
*해설: Splunk는 데이터를 인덱싱할 때 구조를 고정하지 않고, 사용자가 검색(Search)을 수행하는 시점에 데이터를 해석합니다. 이를 통해 비정형 데이터도 자유롭게 분석할 수 있습니다.*

**A3. 정답: 2) 시스템의 모든 활동을 기록한 유일한 증거(DNA)이기 때문에**
*해설: 로그는 보안 사고 발생 시 공격자의 경로를 추적하고 피해 범위를 산정할 수 있는 가장 확실한 디지털 증거입니다.*
